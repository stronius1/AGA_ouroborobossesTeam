
# Глава 2: Введение в анализ данных

## 1.1 Основы анализа

### 1.1.1 Что такое анализ данных
Анализ данных — это процесс преобразования данных в инсайты для принятия решений. Используются методы статистики, машинного обучения и визуализации.

### 1.1.2 Источники данных
- Базы данных
- API
- CSV-файлы
- Web scraping

### 1.1.3 Роль аналитика данных
Аналитик данных:
- Собирает и обрабатывает данные
- Строит визуализации
- Делает выводы и прогнозы

## 1.2 Подходы к анализу

### 1.2.1 Описательный анализ
Используется для описания того, что уже произошло.

**Пример таблицы:**

| Метрика      | Значение |
|--------------|----------|
| Средний доход | 52,000   |
| Медиана       | 48,500   |
| Максимум      | 130,000  |

### 1.2.2 Диагностический анализ
Позволяет понять причины произошедших событий.

**Методы:**
- Корреляционный анализ
- Регрессия
- Анализ аномалий

### 1.2.3 Предсказательный анализ
Прогнозирует будущее на основе прошлых данных.

**Примеры моделей:**
- Линейная регрессия
- Случайный лес
- XGBoost

## 1.3 Инструменты аналитика

### 1.3.1 Excel и Google Sheets
Подходят для базовой аналитики и визуализации.

### 1.3.2 Python
Используется для продвинутой обработки данных:

```python
import pandas as pd

df = pd.read_csv("data.csv")
print(df.describe())
```

### 1.3.3 SQL
Язык для работы с реляционными базами данных.

```sql
SELECT region, AVG(sales) 
FROM orders 
GROUP BY region;
```

# Глава 2: Машинное обучение

## 2.1 Основы

### 2.1.1 Надзорное обучение
Модели обучаются на размеченных данных.

**Примеры задач:**
- Классификация
- Регрессия

### 2.1.2 Безнадзорное обучение
Работа с неразмеченными данными. Часто используется для кластеризации.

**Алгоритмы:**
- K-Means
- DBSCAN

### 2.1.3 Обучение с подкреплением
Модель учится через награды и штрафы.

## 2.2 Оценка моделей

### 2.2.1 Метрики классификации

| Метрика       | Описание                       |
|---------------|--------------------------------|
| Accuracy      | Доля правильных предсказаний   |
| Precision     | Точность положительных ответов |
| Recall        | Полнота положительных ответов  |

### 2.2.2 Кросс-валидация
Метод оценки, при котором данные делятся на несколько подвыборок.

### 2.2.3 Переобучение
Когда модель слишком хорошо запоминает обучающую выборку и плохо работает на новых данных.

## 2.3 Библиотеки

### 2.3.1 Scikit-learn
Стандартная библиотека для ML в Python.

### 2.3.2 TensorFlow
Фреймворк для глубокого обучения от Google.

### 2.3.3 PyTorch
Альтернатива TensorFlow, популярна в научной среде.
