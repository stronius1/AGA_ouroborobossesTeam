# Интеграция с системой продуктовой аналитики ClickStream 📊

Если вы используете SEAF.ArchTool в режиме портала, вы можете подключить сбор аналитики и отправлять данные в свой кабинет в ClickStream. 
Это позволит вам отслеживать использование портала, выявлять узкие места и принимать решения на основе реальных данных.

На текущий момент функциональность доступна только для компаний группы Сбер.

## О ClickStream 📝

**[ClickStream](https://gitverse.ru/clickstream/sdk-info)** - инструмент продуктовой аналитики для получения наглядных отчётов по web или mobile интерфейсам и анализа поведения пользователей.
Инструмент помогает понять, как пользователи взаимодействуют с продуктом, где сильные, а где слабые стороны, а также определить реальный охват пользователей и более точно сформулировать стратегические цели продукта.
ClickStream имеет схожий функционал с Яндекс Метрикой, Firebase, AppsFlyer.

🔒Мы не имеем доступа к данным, которые вы собираете у себя. Все хранится и анализируется только в рамках вашего аккаунта.

## Что нужно для подключения 🔑

1. Зарегистрируйтесь в ClickStream и получите API_KEY и REPORT_URL
2. В .env укажите эти значения в переменных VUE_APP_CLICKSTREAM_API_KEY и VUE_APP_CLICKSTREAM_REPORT_URL соответственно.
3. Готово - события начнут отправляться в ваш кабинет.

## Какие события отправляются и какие отчеты можно из них построить 💡

Автоматически отправляются события Просмотр страницы и Время загрузки.

Событие просмотр страницы содержит параметры:
* fromFullPath - путь страницы, с которой был совершен переход на текущую
* targetFullPath - полный адрес текущей страницы
* loadTimeMs - скорость загрузки страницы в мс
* loadTimeMsRange - скорость загрузки страницы, определенная в диапазон
* platform - платформа 
* componentName - название компонента

Событие время загрузки содержит параметры:
* fromFullPath - путь страницы, с которой был совершен переход на текущую
* targetFullPath - полный адрес текущей страницы
* loadTimeMs - скорость загрузки страницы в мс
* platform - платформа
* componentName - название компонента

Из полученных событий можно построить следующие отчеты:
* MAU (Monthly Active Users) - количество уникальных пользователей за месяц
* DAU (Daily Active Users) - количество уникальных пользователей за день
* Страницы, которые посещал пользователь
* Количество обращений к странице
* Скорость загрузки страниц

_Список событий может изменяться по мере развития продукта. Все события отправляются в обезличенном виде_
