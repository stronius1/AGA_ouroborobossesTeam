# Интеграция AI-агентов с редактируемыми таблицами

## Обзор

Реализована функциональность автоматического заполнения ячеек в редактируемых таблицах с помощью AI-агентов.

## Особенности

1. **Иконка AI в заголовках**: Для колонок с настроенным агентом появляется иконка робота
2. **Инициализация агента**: Агент инициализируется с параметрами из URL
3. **Итеративная обработка**: Каждая строка таблицы обрабатывается отдельно
4. **Автоматическое заполнение**: Ответы от агента записываются в ячейки (если не "None")
5. **Управление сессиями**: Сессия агента автоматически завершается после обработки

## Конфигурация

В headers таблицы добавляется новое поле `agent`:

```yaml
- value: tech_component
  text: Техкомпонент
  editable: true
  save:
    path: mock.yaml
  agent:
    aichat.scenarios.tech_param_prediction:
      params: 
        - domain
      input_headers:
        - title
        - description
        - id
      request: | 
        Предложи технический параметр для системы:
        Идентификатор системы: [id]
        Описание системы: [description]
        Наименование системы [title]
```

### Параметры agent:

- **Ключ**: Идентификатор агента в формате `документ.scenarios.сценарий`
- **params**: Параметры, передаваемые из URL в агент
- **input_headers**: Поля строки, используемые для формирования запроса
- **request**: Шаблон запроса с подстановками в квадратных скобках

## Пример использования

1. Откройте редактируемую таблицу с настроенным агентом
2. Убедитесь, что в URL есть необходимые параметры (например, `?domain=ecogroup.berezka`)
3. Нажмите на иконку AI в заголовке колонки
4. Дождитесь завершения обработки

## Логика работы

1. **Инициализация**: При нажатии на иконку AI извлекается конфигурация агента
2. **Получение агента**: Загружается конфигурация агента из документа
3. **Параметры**: Собираются параметры из URL согласно настройкам
4. **Создание сессии**: Инициализируется новая сессия агента
5. **Обработка строк**: Для каждой строки:
   - Формируется запрос с подстановками из input_headers
   - Отправляется запрос агенту
   - Ответ записывается в ячейку (если не "None")
6. **Завершение**: Сессия агента закрывается

## Требования

- Настроенный GigaChat сервис
- Конфигурированные агенты в документах
- Редактируемые таблицы с save конфигурацией
