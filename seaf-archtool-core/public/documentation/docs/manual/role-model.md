# Ролевая модель и доступы на портале. Инструкция для администраторов
[[toc]]
## Назначение документа

Документ описывает, как на текущий момент работает доступ на портале и какие действия должен выполнить администратор, чтобы пользователь получил доступ к данным.

Функциональность ролей и альтернативных источников доступов будет расширена в будущих версиях.

## Как сейчас работает доступ

На текущий момент:

* роли, передаваемые из IdP, не используются;
* доступ пользователя определяется доступами в Bitbucket.

Если пользователь не видит данные на портале — причину следует искать в настройках доступа в Bitbucket.

## Источник прав доступа

### Bitbucket

Bitbucket является единственным источником прав доступа для портала.

Портал:

* использует данные о доступах пользователя к репозиториям Bitbucket;
* сопоставляет их с учетной записью пользователя из токена IdP.

Важно:

* учетная запись в Bitbucket должна совпадать с учетной записью, используемой для входа в портал (один и тот же IdP);
* если используются разные учетные записи — доступ на портале определен не будет.

## Поддерживаемые роли

На текущий момент портал не выполняет проверок ролей.

Роли, передаваемые из IdP:

* принимаются порталом;
* сохраняются в контексте пользователя;
* не участвуют в принятии решений о доступе.

Портал корректно работает с любыми ролями (включая пользовательские), переданными в токене IdP.

Фактический доступ пользователя определяется доступами в Bitbucket.

## Наделение пользователя доступом

### Предоставление доступа

Для того чтобы пользователь получил доступ к данным на портале, необходимо:

1. Определить репозиторий Bitbucket, соответствующий нужным данным.
2. Убедиться, что пользователь:

    * существует в Bitbucket;
    * использует ту же учетную запись / IdP, что и для входа в портал.
3. Выдать пользователю доступ к репозиторию Bitbucket:

    * *Read* — для просмотра данных;
    * более высокий уровень — для _будущих_ операций записи;
4. Дождаться применения изменений (происходит автоматически).

После этого пользователь сможет видеть соответствующие данные на портале.

### Отзыв доступа

Для отзыва доступа необходимо:

1. Удалить пользователя из списка участников репозитория Bitbucket или
2. Понизить уровень его прав.

Изменения вступят в силу автоматически.

## Развитие ролевой модели

В будущих версиях планируется использование ролей для ограничения доступа к разделам портала.
