Познакомимся с основными типами данных, с которыми таблица умеет работать

Типы данных в ячейках настраиваются через заголовки столбцов

# text (значение по умолчанию)

## Описание

Отображает содержимое с текстом

## Формат ожидаемых данных

- Строка
- Число
- Прочие приводимые значения

# checkbox

## Описание

Отображает чекбокс

## Формат ожидаемых данных

- true
- false
- Прочие приводимые значения

# link

## Описание

Отображает кликабельную ссылку. Поддерживаются как внешние, так и внутренние переходы.
Ссылка остается кликабельной, если в качестве данных для ячейки с типом `link` переданы:
- валидный url
- внутренний путь, начинающийся с `/`

В ином случае, данные отображаются обычным текстом

## Формат ожидаемых данных

- Строка (URL)
- Объект, с обязательным свойством `href` (ссылка для перехода) и опциональным `text` (отображаемое значение)

```yaml
  {
      "href": "https://www.google.ru/",
      "text": "google"
  }
```

- Массив объектов

```yaml
[
  { "href": "https://www.google.ru/", "text": "Google" },
  { "href": "https://ya.ru/", "text": "Yandex" },
]
```

# fn

JSONata-выражение.
Результат выполнения которого будет записан в ячейку.
Если при вычислении JSONata выражения произойдет ошибка, в ячейке будет выведено "Ошибка"
В выражение в качестве дополнительного параметра передается параметр `row` (Зарезервированное имя), содержащий актуальные данные из текущей строки

# select / multiple-select

## Описание

`select` - Выпадающий список
`multiple-select` - Выпадающий список с возможностью выбора нескольких значений

## Дополнительные свойства

### options (обязательное к заполнению)

Список доступных для выбора опций

Ожидаемый формат данных:
массив объектов, с обязательным полем `value` (идентификатор) и опциональным `text` (значение отражаемое в таблице)

```yaml
options:
  - value: vanilla
    text: ванильное
  - value: chocolate
    text: шоколадное
  - value: milk
```

### option_id (обязательное при формировании списка по условию)

См. `Дополнительные возможности/Выпадающий список по условию`

## Дополнительные возможности

### Выпадающий список по условию

Позволяет отобразить разные списки в зависимости от каких-либо условий
Для формирования такого списка необходимо изменить формат данных в `options` и добавить дополнительное свойство `option_id`

Ожидаемый формат данных для `options`:
объект, где ключ - идентификатор для выпадающего списка и идентификатор одного из заголовков, а значение - массив объектов (рассмотрен выше)

`option_id` - идентификатор-ссылка на один из заголовков в таблице и на одну из групп опций

В примере ниже, таблица из 2 колонок (`pet` и `breed`). В зависимости от выборки в колонке `pet` для колонки `breed` будут отражаться разные выпадающие списки

```yaml
headers:
  # Выпадающий список по условию (колонки pet и breed)
  - value: pet
    text: Питомец
    width: 150px
    type: select
    options:
      - value: cat
        text: Кошка
      - value: dog
        text: Собака

  - value: breed
    text: Порода
    width: 150px
    type: multiple-select
    option_id: pet
    # option_id - идентификатор-ссылка на колонку в таблице
    # позволяет настроить опциональные значения в выпадающем списке
    # Обязательное когда options задан объектом
    options:
      cat:
        # Опция применяется если значение из колонки "pet" для текущей строки равно "cat"
        - value: siamese
          text: сиамская
        - value: siberian
          text: сибирская
        - value: maine_coon
          text: мейн-кун
      dog:
        # Опция применяется если значение из колонки "pet" для текущей строки равно "dog"
        - value: mastiff
          text: мастиф
        - value: dalmatian
          text: далматинец
        - value: pug
          text: мопс
```

# Пример использования и описание каждого заголовка

```yaml
headers:
  # headers - Массив заголовков
  # Обязательное
  - value: ID
    # value - Идентификатор поля для вывода в колонке
    # Обязательное
    type: text
    # type - Тип вывода ячеек для указанной колонки
    # Дефолтное значение:  text
    # Допустимые значения: text / select / multiple-select / checkbox / link / fn

  - value: name
    text: Имя
    # text - Отображаемый заголовок
    # Дефолтное значение: подставляется из значения value

  - value: age
    text: Возраст

  - value: is_adult
    text: Возрастная группа
    width: 150px
    type: fn
    # type: fn позволяет вывести произвольные данные
    # рассчитанные JSONata-выражением, на основе прочих данных в строке
    fn: >
      (
          $age := $number(row.age);

          $reduce_cb := function($acc, $group){(
            $acc = null
            ? $age > $group.end
              ? null
              : $group."group_name"
            : $acc
          )};

          row.age 
              ? $reduce(age_groups, $reduce_cb, null)
              : "Возраст не указан"
      )
  # fn - JSONata выражение
  # Результат выражения отражается в ячейке
  # Если при вычислении JSONata выражения произойдет ошибка, в ячейке будет выведено "Ошибка"
  # Данные из строки доступны как параметр (origin) "row" (зарезервированное имя)

  # Параметры для текущего примера:
  #   $ = {
  #         row: {...},                # Зарезервированное имя (origin), содержит данные из строки
  #         body: {...},               # origin: body
  #         color_options: {...},      # origin: color_options
  #         age_groups: {...}          # origin: age_groups
  #       }

  - value: subscription
    text: Подписка
    width: 80px
    type: checkbox

  - value: ice_cream
    text: Мороженное
    width: 100px
    type: select
    option_id: ice_cream
    # option_id - идентификатор-ссылка на колонку в таблице
    # позволяет настроить опциональные значения в выпадающем списке
    # Обязательное когда options задан объектом
    options:
      # options - опции для типа "select" и "multiple-select"
      # Обязательное для "select" и "multiple-select"
      # Допустимые значения:
      # 1. массив объектов, с обязательным полем `value` (идентификатор) и опциональным `text` (значение отображаемое в таблице)
      # 2. объект, где ключ - идентификатор для выпадающего списка, значение - массив объектов, с обязательным полем `value` (идентификатор) и опциональным `text` (значение отображаемое в таблице)
      # также значением может быть JSONata выражение (или origin), выполнение которого вернет аналогичную структуру
      - value: vanilla
        text: ванильное
      - value: chocolate
        text: шоколадное

  - value: color
    text: Цвет
    width: 130px
    type: multiple-select
    options: >
      (color_options)

  # Выпадающий список по условию (колонки pet и breed)
  - value: pet
    text: Питомец
    width: 150px
    type: select
    options:
      - value: cat
        text: Кошка
      - value: dog
        text: Собака

  - value: breed
    text: Порода
    width: 150px
    type: multiple-select
    option_id: pet
    # option_id - идентификатор-ссылка на колонку в таблице
    # позволяет настроить опциональные значения в выпадающем списке
    # Обязательное когда options задан объектом
    options:
      cat:
        # Опция применяется если значение из колонки "pet" для текущей строки равно "cat"
        - value: siamese
          text: сиамская
        - value: siberian
          text: сибирская
        - value: maine_coon
          text: мейн-кун
      dog:
        # Опция применяется если значение из колонки "pet" для текущей строки равно "dog"
        - value: mastiff
          text: мастиф
        - value: dalmatian
          text: далматинец
        - value: pug
          text: мопс

  - value: link
    width: 150px
    text: Ссылки
    type: link
    # Допустимые значения для ссылок:
    # 1. Текст:
    #   https://www.google.ru/
    # 2. Объект:
    #   {
    #     "href": "/architect/components/system.auth",
    #     "text": "Авторизация"
    #   }
    # 3. Массив объектов:
    #   [
    #       {
    #           "href": "https://www.google.ru/",
    #           "text": "google"
    #       },
    #       {
    #           "href": "/docs/example",
    #           "text": "Пример документа"
    #       }
    #   ]
```
