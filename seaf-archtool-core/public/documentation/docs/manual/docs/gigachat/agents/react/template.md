# `ReAct` - агент

# Описание

Агент позволяет взаимодействовать с GigaChat с использованием дополнительных инструментов/тулов (tool).

# ⚠️ Доступность ⚠️

Агент доступен в:

- `backend`-mode

# Знакомство

Познакомимся с агентом, опишем структуру документа:

```yaml
docs:
  archtool.plugins.gigachat.agents:
    type: ai-chat
    scenarios:
      calculator:
        title: Калькулятор
        type: react
        model: GigaChat-2
        system_prompt: |
          Ты калькулятор осуществляющий расчет по следующей функции:
          x + y - z.

          Значение [z] ты получаешь от пользователя.
          Если пользователь передал не число - верни ответ: "Пожалуйста, введите число"

          Для получения прочих значений:
           - Используй инструмент get_x для получения значения [x]
           - Используй инструмент get_y для получения значения [y]
           - Используй инструмент calculate для расчета функции

          Верни в ответ полученное число.
          На каждый новый запрос пользователя получай значения [x] и [y] повторно, используя соответсвующие инструменты.

        tools:
          - name: get_x
            description: Получает значение x.
            query: >
              (
                100
              )
          - name: get_y
            description: Получает значение y.
            query: >
              (
                50
              )
          - name: calculate
            description: Выполняет расчет.
            schema:
              type: object
              properties:
                x:
                  title: Значение переменной x
                  type: number
                y:
                  title: Значение переменной y
                  type: number
                z:
                  title: Значение переменной z
                  type: number
            query: >
              (
                  $number('${x}') + $number('${y}') - $number('${z}')
              )
```

# Базовое описание

## `system_prompt`

Основной промпт

## `model`

Название модели, от которой нужно получить ответ

## `temperature`

Необязательный параметр.

Температура выборки в диапазоне от ноля до двух (число).

## `topP`

Необязательный параметр.

Альтернатива параметру температуры (число).

## `tools`

Список тулов

Рассмотрим описание тула

### `name`

Идентификатор тула

### `description`

Описание тула для llm. Должен содержать информацию о способе применения, в контексте общей задачи.

### `schema`

Описание json-схемы.
Если описана схема - будет произведена подстановка описанных значений в JSONata-выражение описанное `query`.

### `query`

JSONata-выражение, выполняемое для исполнения тула

JSONata выражение, применится к озеру данных. Добавляет результат выполнения выражения, в качестве контекста, к основному промпту.

# Пример формирования запроса

Рассмотрим механику формирования запроса и работы тулов на примере.

В качестве системного промпта используется значение переданное в поле `system_prompt`.
Исходя из задачи в промпте, ассистент определит необходимые шаги для ее достижения.
Выполнит необходимые тулы и предоставит ответ.

В нашем примере ассистент:

- Инициирует и подготовит все тулы.
- Подготовит системный промпт из поля `system_prompt`.
- Воспользуется тулом `get_x` для получения значения `x`.
- Воспользуется тулом `get_y` для получения значения `y`.
- Воспользуется тулом `calculate` для выполнения расчета. В туле указана json-схема, т.о.:
- - в выражение описанное в `query` будут подставлены указанные параметры.
- - ассистент выполнит JSONata-выражение с обновленным значением.
- После получения всех значений ассистент подготовит ответ и вернет пользователю.
