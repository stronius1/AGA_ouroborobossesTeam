## v2026.29.0

### Новое и улучшения
- В репозиторий проекта добавлены Docker и HCL файлы (для Docker Compose), а также документация по их использованию. Это стандартизирует и упрощает процесс локального развертывания и разработки.
- Реализован базовый механизм для приема архкода из плагина в сервис для публикации в Bitbucket. Это закладывает фундамент для будущей публикации через API.
- Изменена схема взаимодействия сервисов, добавлена оркестрация тяжелых процессов. Релоад портала и обновление прав из битбакет теперь выполняются оркестратором в асинхронном режиме.
- Интеграция с Draw.io:
    - Обновлены шаблоны и расширена палитра компонентов для более удобного моделирования.
    - Реализована прямая запись изменений из Draw.io в архитектурное озеро данных.
    - Доработан механизм раскладки компонентов технической архитектуры на схеме Draw.io.
    - Реализована дедупликация связей провайдеров и улучшено распределение объектов по слоям для более чистой и управляемой визуализации.
    - Расширено логирование плагина Draw.io для упрощения диагностики и отладки.
- ИИ
    - Добавлена возможность вызова одного ИИ-агента другим агентом для решения сложных, составных задач.
    - Реализована запись изменений в архитектурное озеро на портале с использованием ИИ.
    - Добавлена поддержка протокола Model Context Protocol (MCP).
    - Реализована суммаризация диалога, что позволило значительно снизить потребление токенов при работе с большими контекстами.
- Выполнена миграция фронтенд-части на Vue3. Это закладывает основу для повышения производительности, стабильности и использования современных технологий в пользовательском интерфейсе.
- Переработана сборка и упаковка фронтовых компонент для плагина IDE, что стандартизирует процесс и повышает надежность сборки.
### Исправлены ошибки
- Усилена защита системных промптов:
    - добавлена специальная метка безопасности (canary token);
    - реализовано повторение ключевых инструкций в точке завершения пользовательского ввода; 
    - введена возможность однозначного определения авторства pull request'ов;
    - добавлена валидация и ограничение по длине для пути вывода (output_path).
### Дополнительно
- Из системы удален устаревший компонент «Техрадар».
- Добавлена документация по конвертации моделей (seaf2drawio).
- Обновлен логотип в компоненте AIGenA.

## v3.27.0

### Новое и улучшения

- В excel-export:
  * при сохранении файла расширение больше не дублируется, если оно уже указано пользователем в имени;
  * при формировании имени файла с помощью JSONata введено ограничение: длина имени не может превышать 255 символов. При превышении лимита система оображает ошибку, предотвращая некорректное создание файла.

  * файлы, сохранённые через Р7-офис, импортируются без ошибок.

## v3.25.0

### Новое и улучшения
#### Обновление документации и повышение прозрачности

- Появилась настройка лимита длины сохраняемых строк в кэше, позволяющая администратору гибко контролировать использование ресурсов. Инструкция по настройке доступна в example.env.

- Теперь можно указать новый адрес сервиса *PlantUML* без пересборки приложения. Инструкция по настройке доступна в example.env.

- Алгоритм *seaf2drawio* обеспечивает интеллектуальное распределение сетевых сервисов по группам внутри сегмента.
#### Расширенные возможности импорта и экспорта
- Поддержаны импорт и экспорт в *xls* с возможностью динамического задания имени файла.
- При импорте из *xls* в плагин появилась настройка способа отображения пустых ячеек в *YAML*-структуре.
- Экспорт в *xls* теперь учитывает активные фильтры, выгружая только выбранные строки из редактируемых таблиц.
- Добавлена функция удаления строк прямо в интерфейсе редактируемых таблиц. Инструкция по настройке удаления добавлена в документацию по базовой настройке таблиц.
#### JSONata
- Добавлена функция $tryexecute(), которая позволяет продолжать выполнение JSONata-запросов даже при возникновении ошибки внутри функции, обернутой в эту функцию.
