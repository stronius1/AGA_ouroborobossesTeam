# Агенты (scenarios)

Свойство `scenarios` содежит описание подключаемых агентов

# Общие свойства описания агентов

Перед тем как рассмотреть каждый агент детально, рассмотрим общие свойства описания агентов:

```yaml
archtool.plugins.gigachat.example:
  type: ai-chat
  scenarios:
    scenario_id:
      title: Консультант
      type: simple
      history: true
      # ... other props
```

## `title`

**Описание**: Имя (placeholder) агента, отражаемое в меню в нижней части экрана.

**Допустимые значения**: строка.

**Обязательное**: да.

## `type`

**Описание**: Идентификатор типа агента.

**Допустимые значения**: [Описаны ниже](#доступные-типы-агентов).

**Обязательное**: да.

## `history`

**Описание**: Флаг, позволяет отключить передачу истории сообщений между запросами, уменьшив потребление токенов.
Отключение будет полезно на больших текстовых объемах, где нет необходимости в контексте всех ранее отправленных сообщений в диалоге.

**Допустимые значения**: Boolean.

**Значение по умолчанию**: `true`.

**Обязательное**: нет.

# Доступные типы агентов:

## [`simple`](/entities/docs/blank?dh-doc-id=archtool.plugins.gigachat.agents.simple.menu)

Взаимодействует c API GigaChat.
Позволяет описать промпт и добавить в него результат выполнения JSONata-выражения в качестве контекста.

## [`react`](/entities/docs/blank?dh-doc-id=archtool.plugins.gigachat.agents.react.menu)

Взаимодействует c API GigaChat.
Позволяет использовать тулы (function-calling инструменты) для предоставления ответа.

## [`aigena`](/entities/docs/blank?dh-doc-id=archtool.plugins.gigachat.agents.aigena.menu)

Взаимодействует с API AiGenA.
Позволяет получить консультацию по стандартам ДЗО.
