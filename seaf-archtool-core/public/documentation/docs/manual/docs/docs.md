# Документы (docs)

## Доступные документы

Документ - многофункциональная сущность. Поддерживаются форматы:
* **[AsyncAPI](/docs/archtool.asyncapi)** - язык для описания событийно-ориентированных сервисов. [Подробнее](https://www.asyncapi.com/).
* **[Swagger (OpenAPI)](/docs/archtool.swagger)**  - язык описания интерфейсов для описания RESTful API, выраженных с помощью JSON. [Подробнее](https://swagger.io/).
* **[BPMN](/docs/archtool.bpmn)** - нотация для описания и моделирования бизнес-процессов. [Подробнее](http://bpmn.io).
* **[PlantUML](/docs/archtool.plantuml)** - позволяет создавать диаграммы из обычного текстового языка. [Подробнее](https://plantuml.com/ru/).
* **[Mermaid](/docs/archtool.mermaid)** - Аналогично PlantUML позволяет создавать диаграммы из текста. [Подробнее](https://mermaid-js.github.io/mermaid/#/).
* **[Draw.io](/docs/archtool.plugins.drawio)** - инструмент для создания и редактирования диаграмм любой сложности в визуальном редакторе. [Подробнее](https://www.drawio.com/).
* **[SmartAnts](/docs/archtool.smartants)** - продвинутый инструмент презентации архитектуры в виде диаграмм с широкими возможностями.
* **[Network](/docs/archtool.network)** - сетевые диаграммы.
* **[Диаграммы](/docs/archtool.charts)** - включает разные типы графиков для наглядного сравнения и анализа данных.
* **[Markdown](/docs/archtool.markdown)** - облегчённый язык разметки, созданный с целью обозначения форматирования в простом тексте. [Подробнее](https://ru.wikipedia.org/wiki/Markdown).
* **[Компоновщик (markaper)](/docs/archtool.markaper)** - компонент для организации интерфейса, позволяющий создавать табы выпадающие списки и другие элементы управления на странице.
* **[HTML](/docs/archtool.plugins.example)** - язык разметки для структурирования и отображения контента. [Подробнее](https://ru.wikipedia.org/wiki/HTML).
* **[SVG](/docs/archtool.plugins.svg_inline)** - язык разметки для создания масштабируемой векторной графики. [Подробнее](https://ru.wikipedia.org/wiki/SVG).
* **[Таблица](/docs/archtool.tables)** - документ для представления данных в табличной форме.
* **[Редактируемая таблица](/docs/archtool.plugins.editable-table.menu)** - документ для представления данных в табличной форме, который позволяет редактировать данные через интерфейс.
* **[DevTool](/docs/archtool.plugins.devtool_new)** - инструмент для обращения к данным архитектуры. Расширение JSONata.
* **[GigaChat](/docs/archtool.plugins.gigachat.menu)** - AI-инструмент для общения и генерации контента в диалоговом формате. [Подробнее](https://giga.chat/).

Перечень доступных документов можно самостоятельно расширять за счет создания [плагинов](/docs/archtool.plugins.intro). 

## Использование

Описание документов в манифесте:
```yaml
docs:                                 # Документы
  archtool.manual:                    # Идентификатор документа
        icon: lightbulb_outline       # Иконка, которы будет отображаться в дереве навигации
        location: SEAF.ArchTool/Руководство  # Размещение документа в меню (если требуется отражать)
                                      # Если в начале строки указывается "/" документ будет размещен от корня,
                                      # в противном случае, он разместится в определенном конфигурацией пункте меню.
                                      # Подробнее ниже.
        description: Пользовательское руководство SEAF.ArchTool  # Краткое описание сути документа (опиционально)
        type: markdown                # Тип документа (OpenAPI / markdown / PlantUML / Table)
        subjects:                     # К какому архитектурному объекту документ имеет отношение
            - archtool.front            # Идентификатор архитектурного объекта
        source: ../docs/manual.md     # Путь к документу 
        data: ...                     # Данные для документа (см. таблицы)  
        origin: ...                   # Исходные данные (см. таблицы)
```

## Разметка документов на объекты

Документы могут иметь связи с архитектурными объектами. Для этого в секции "subjects" необходимо перечислить их
идентификаторы. Если объект связан с документом, в его карточке он будет отображаться. Например, для объекта
**archtool.front** карточка выглядит так:


![Карточка компонента](@component/archtool.front)

В разделе "Документы" можно найти ссылки на связанные с объектом документы.

## Конфигурирование

Поведение документов можно конфигурировать через их сущность. Для этого необходимо переопределить 
параметры по умолчанию.

```yaml
...
entities:                             # Обращаемся к сущностям  
  docs:                               # К документам
    config:                           # Здесь хранится конфигурация по умолчанию
      root_menu: Документы            # Корневой элемент меню для документов по умолчанию
                                      # Значение может определять структуру через "/". В конце символ "/" не допускается.
                                      # Например: Аналитика/Документация
...
```
