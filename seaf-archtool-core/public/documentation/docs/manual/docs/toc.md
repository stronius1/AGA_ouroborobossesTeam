
# Отключение автооглавления в Markdown-документах

В Markdown добавлена возможность отключения автоматического оглавления (TOC), а также гибкого размещения его в тексте.

## Правила гибкого размещения TOC в тексте
- TOC должно вставляться в то место, где автор указал маркер **[[toc]]**.
- Если маркер указан — TOC рендерится только в этом месте.
- Если маркера нет, но TOC включено — вставляется в начало документа.
- Если маркера два в документе, то TOC вставляется только в первый.
- Если в YAML **toc: false**, при этом в самом документе есть маркер **[[toc]]**, то не добавляем заголовок.
- Если в YAML **toc: true**, тогда TOC рендерится даже если заголовков мало (в начале документа или на месте маркера **[[toc]]** при его наличии).

## Автоматическое отключение TOC для документов с количеством заголовков менее 3
- При наличии 1-3 заголовков TOC автоматически не вставляется (по умолчанию).
- Если заголовков более 3, то наличие заголовков выставляется по режиму, который настроил сам пользователь в настройках проекта для всего документа.
- **Нет зависимости от уровня вложенности заголовка, важно их количество**.

## Примеры использования гибкой настройки в Markdown-документах
1. Markdown-документ, содержащий 3 заголовка и более отображается с включенным автооглавлением по умолчанию, если пользователем не выбрано иное:

```yaml
  ...
  seaf.no-toc:
    type: markdown
    location: SEAF.ArchTool/Руководство/Документы/Markdown/02 Отключение и перенос автооглавления/021 Пример TOC (по умолчанию)
    source: test.md
  ...
```

[Результат отображения Markdown-документа c TOC (по умолчанию)](/entities/docs/blank?dh-doc-id=seaf.no-toc)

2. Применив к этому документу флаг toc:false пользователь принудительно выключит автооглавление:

```yaml
  ...
  seaf.toc-false:
    type: markdown
    toc: false
    location: SEAF.ArchTool/Руководство/Документы/Markdown/02 Отключение и перенос автооглавления/022 Пример TOC с toc:false
    source: test.md
  ...
```

[Результат отображения Markdown-документа с toc:false](/entities/docs/blank?dh-doc-id=seaf.toc-false)

3. В Markdown-документе с количестом заголовков менее 3 по умолчанию автооглавление отключено:

```yaml
  ...
  seaf.two-header:
    type: markdown
    location: SEAF.ArchTool/Руководство/Документы/Markdown/02 Отключение и перенос автооглавления/023 Пример с 2мя заголовками (по умолчанию)
    source: 2header.md
  ...
```

[Результат отображения Markdown-документа с 2мя заголовками (по умолчанию)](/entities/docs/blank?dh-doc-id=seaf.two-header)

4. В Markdown-документе с количестом заголовков менее 3 пользователь также может принудительно включить его:

```yaml
  ...
  seaf.toc-true:
    type: markdown
    toc: true
    location: SEAF.ArchTool/Руководство/Документы/Markdown/02 Отключение и перенос автооглавления/024 Пример с 2мя заголовками и toc:true
    source: 2header.md
  ...
```

[Результат отображения Markdown-документа с 2мя заголовками и toc:true](/entities/docs/blank?dh-doc-id=seaf.toc-true)

5. Маркер **[[toc]]** позволяет указать место размещения оглавления:

```
### 1.1.2 Источники данных
- Базы данных
- API
- CSV-файлы
- Web scraping

Оглавление:

[[toc]]

### 1.1.3 Роль аналитика данных
Аналитик данных:
- Собирает и обрабатывает данные
- Строит визуализации
- Делает выводы и прогнозы
```

[Результат отображения Markdown-документа с применением маркера [[toc]]](/entities/docs/blank?dh-doc-id=seaf.custom-toc)

6. В случае использования маркера **[[toc]]** и **toc:false** флаг имеет высший приоритет:

```yaml
  ...
  seaf.custom-toc-and-toc-false:
    type: markdown
    toc: false
    location: SEAF.ArchTool/Руководство/Документы/Markdown/02 Отключение и перенос автооглавления/026 Пример с маркером [[toc]] и toc:false
    source: custom-toc-place.md
  ...
```
