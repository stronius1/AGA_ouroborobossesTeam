# ADR-0018: Брокер сообщений для событий антифрода

## Контекст

Антифрод-платформе нужны упорядоченные события с повторной доставкой и
наблюдаемым отставанием потребителей. Интеграционная шина остаётся точкой
управления контрактами.

## Решение

Использовать корпоративный кластер Kafka за интеграционной шиной AS-0008;
AS-0012 получает события из выделенного топика с версионируемой схемой.

## Альтернативы

Рассматривались RabbitMQ и синхронный REST. RabbitMQ не выбран из-за модели
долгого хранения, REST — из-за жёсткой временной связанности участников.

## Последствия

Потребуются реестр схем, мониторинг consumer lag и регламент повторной
обработки сообщений.
